# 移动端优化说明

## 改进内容

针对移动端用户体验进行了优化，现在在手机上访问页面列表时会有更好的浏览体验。

## 主要变化

### 1. **桌面端（宽度 > 768px）**
- ✅ 保持原有的左右分栏布局
- ✅ 左侧显示页面列表
- ✅ 右侧 iframe 预览选中页面
- ✅ 支持刷新和新标签打开

### 2. **移动端（宽度 ≤ 768px）**
- ✅ **全屏列表模式**：只显示文章标题列表，占满整个屏幕
- ✅ **点击即预览**：点击标题后，在当前页面全屏打开预览
- ✅ **全屏预览模式**：
  - 顶部显示页面标题和操作按钮
  - 中间区域全屏显示页面内容
  - 右上角"↗"按钮：在新标签页打开
  - "✕ 关闭"按钮：返回列表
- ✅ **流畅切换**：预览和列表之间平滑过渡
- ✅ **禁止背景滚动**：预览时锁定背景，避免误操作

## 技术实现

### CSS 响应式设计

```css
@media (max-width: 768px) {
    /* 隐藏桌面端预览区 */
    .preview-area {
        display: none;
    }
    
    /* 移动端全屏预览容器 */
    .mobile-preview {
        position: fixed;
        top: 0;
        left: 0;
        right: 0;
        bottom: 0;
        z-index: 1000;
        display: none;
    }
    
    .mobile-preview.active {
        display: flex;
    }
}
```

### JavaScript 智能检测

```javascript
// 检测设备类型
const isMobile = window.innerWidth <= 768;

if (isMobile) {
    // 移动端：全屏预览
    showMobilePreview(page);
} else {
    // 桌面端：右侧预览
    showDesktopPreview(page);
}
```

### 移动端预览功能

1. **显示预览**：
   - 创建全屏覆盖层
   - 加载 iframe 内容
   - 禁止背景滚动

2. **关闭预览**：
   - 隐藏覆盖层
   - 恢复背景滚动
   - 清空 iframe（节省资源）

## 用户体验提升

### 之前的问题
- ❌ 移动端左右分栏，空间狭小
- ❌ iframe 预览区域太小，难以阅读
- ❌ 需要频繁滚动查看内容

### 现在的优势
- ✅ 列表全屏显示，充分利用屏幕空间
- ✅ 点击后全屏预览，阅读体验极佳
- ✅ 操作简单直观，符合移动端习惯
- ✅ 一键返回，快速浏览多个页面

## 使用示例

### 手机端操作流程

1. **打开页面列表**
   ```
   http://localhost:8080/pagelist
   ```

2. **浏览列表**
   - 上下滑动查看所有页面
   - 每个页面显示标题和创建时间

3. **查看页面**
   - 点击任意标题
   - 自动进入全屏预览模式

4. **操作选项**
   - 点击右上角"↗"：在新标签页打开
   - 点击"✕ 关闭"：返回列表继续浏览

5. **连续浏览**
   - 关闭预览后，点击下一个标题
   - 无缝切换，流畅体验

## 性能优化

1. **按需加载**：仅在点击时加载 iframe 内容
2. **资源释放**：关闭预览时清空 iframe
3. **流畅动画**：使用 CSS transition 实现平滑过渡
4. **内存管理**：及时清理不需要的 DOM 元素
