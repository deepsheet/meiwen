# 页面列表功能说明

## 功能概述

页面列表功能提供了一个直观的界面，用于浏览和管理最新创建的网页。用户可以通过该页面查看最新的50个网页，并在右侧预览区域实时查看页面内容。

## 访问地址

```
http://localhost:8080/pagelist
```

## 主要功能

### 1. 页面列表展示
- **左侧边栏**：显示最新创建的50个网页
- **排序方式**：按创建时间倒序排列（最新的在前）
- **显示信息**：
  - 页面标题
  - 创建时间（格式：YYYY-MM-DD HH:MM:SS）
  - 新标签打开按钮（↗ 图标）

### 2. 页面预览
- **右侧预览区**：点击左侧列表项后，在 iframe 中实时预览页面内容
- **预览控制**：
  - 刷新按钮：重新加载当前预览的页面
  - 新标签打开按钮：在新标签页中打开完整页面

### 3. 交互特性
- **点击列表项**：选中并预览对应页面
- **悬停效果**：列表项悬停时显示阴影和位移动画
- **激活状态**：当前选中的列表项高亮显示
- **响应式设计**：支持移动端自适应布局

## 技术实现

### 后端接口

#### 1. 页面路由
```python
@app.route('/pagelist', methods=['GET'])
def pagelist_page():
    """返回页面列表页"""
    increment_pv('isheetpv')
    return render_template('pagelist.html')
```

#### 2. API 接口
```python
@app.route('/api/latest-pages', methods=['GET'])
def get_latest_pages_api():
    """获取最新创建的网页列表 API"""
    limit = request.args.get('limit', 50, type=int)
    pages = get_latest_pages(limit)
    return jsonify({
        "status": "success",
        "count": len(pages),
        "pages": pages
    })
```

#### 3. 数据获取函数
```python
def get_latest_pages(limit=50):
    """
    获取最新创建的网页列表
    
    参数:
        limit: 返回的页面数量（默认50）
```

### 前端实现

#### 核心功能
1. **动态加载**：使用 Fetch API 异步获取页面列表数据
2. **实时预览**：通过 iframe 嵌入目标页面
3. **智能标题提取**：自动从文件名中提取有意义的标题
4. **URL 映射**：支持短 URL（/p/{article_id}）和完整路径

#### UI 组件
- **侧边栏**：固定宽度 400px，可滚动列表
- **预览区**：自适应剩余空间
- **空状态提示**：未选择页面时显示引导信息
- **加载状态**：数据加载时显示旋转动画

## 使用示例

### 1. 访问页面列表
在浏览器中打开：`http://localhost:8080/pagelist`

### 2. 预览页面
- 点击左侧任意页面标题
- 右侧自动加载页面预览
- 使用刷新按钮重新加载

### 3. 新标签打开
- 点击列表项右上角的 ↗ 图标
- 或点击预览区顶部的"新标签打开"按钮

## 注意事项

1. **性能优化**：
   - 默认限制为 50 个页面，避免加载过多数据
   - 使用懒加载策略，仅在需要时加载 iframe 内容

2. **安全性**：
   - 所有 URL 都经过验证和转义
   - 防止 XSS 攻击

3. **兼容性**：
   - 支持现代浏览器（Chrome、Firefox、Safari、Edge）
   - 移动端自适应布局

4. **PV 统计**：
   - 每次访问 pagelist 页面都会增加 PV 计数
   - 便于统计页面使用情况
