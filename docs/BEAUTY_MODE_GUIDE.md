# 美化模式选择功能说明

## 功能概述

在文章美化工具（`/static/formatter.html`）中新增了三款美化模式选项，用户可以根据内容类型和需求选择最合适的处理方式。

## 三款美化模式

### 1. 🔒 严格按照原文（默认）

**特点：**
- ✅ 一字不差地保持原文 100% 完整
- ✅ 仅修正错别字、语法错误、标点符号错误
- ✅ 使用丰富的视觉效果呈现（渐变色、卡片、emoji 等）
- ❌ 绝对禁止添加任何原文没有的内容

**适用场景：**
- 正式文档格式化（合同、协议、公文）
- 法律文件
- 新闻报道
- 用户明确要求保持原文不变的情况
- 需要精确引用原文的场合

---

### 2. 📝 适度解读优化

**特点：**
- ✅ 保持原文核心信息、数据、观点完整
- ✅ 可以调整段落顺序、优化表达逻辑
- ✅ 可以添加必要的过渡句、解释性文字
- ✅ 可以修正错别字、语法错误
- ❌ 禁止创造原文没有的新信息、新观点、新数据
- ❌ 禁止添加冗长的引言、结语、总结

**适用场景：**
- 技术文档优化
- 内容逻辑重组
- 增加可读性但不改变原意
- 需要让内容更流畅易懂
- 原文结构较为松散的情况

---

### 3. ✨ 合理扩写丰富

**特点：**
- ✅ 保持原文核心信息、数据、观点完整
- ✅ 可以补充背景信息、案例、数据支撑
- ✅ 可以对专业术语、复杂概念进行解释说明
- ✅ 可以调整段落顺序、添加小标题、优化逻辑流程
- ✅ 可以修正错别字、语法错误
- ❌ 禁止改变原文的核心观点和立场
- ❌ 禁止添加与原文主题无关的冗余内容

**适用场景：**
- 内容营销文章
- 科普文章
- 需要丰富内容的短文
- 需要增加背景说明的专业内容
- 希望内容更加充实、有深度的场合

---

## 使用方法

### 前端界面

1. 打开美化工具页面：`http://localhost:8009/formatter.html`
2. 在"美化模式"区域选择三种模式之一
3. 输入或粘贴文章内容
4. （可选）填写文章标题
5. 点击"🎨 开始美化"按钮

### 数据传输

前端通过 POST 请求将选中的模式传递到后端：

```javascript
{
    "title": "文章标题",
    "content": "文章内容...",
    "content_strategy": "strict"  // 或 "interpret" 或 "expand"
}
```

### 后端处理

后端 API 接收到参数后，传递给 `HTMLFormatter` 服务：

```python
@app.route('/api/format-html', methods=['POST'])
def format_html_api():
    data = request.get_json()
    content_strategy = data.get('content_strategy', 'strict')  # 默认为 strict
    
    formatter = HTMLFormatter(base_url=base_url, model_name=CURRENT_MODEL)
    formatted_html = formatter.format_article(
        content, 
        title, 
        content_strategy=content_strategy
    )
```

最终调用 `LLMClient` 的 `format_article` 方法，该方法会根据不同的策略生成不同的提示词。

## 技术实现

### 前端修改

**文件：** `/static/formatter.html`

添加了美化模式选择器 UI，包含三个单选按钮，默认选中"严格按照原文"。

**文件：** `/static/css/formater.css`

添加了 mode-selector 相关样式，支持响应式布局，在移动端自动切换为单列显示。

**文件：** `/static/js/formatter.js`

在 `formatHTML()` 函数中获取选中的模式值，并通过 `content_strategy` 字段传递到后端。

### 后端修改

**文件：** `src/web_server.py`

在 `/api/format-html` 接口中接收 `content_strategy` 参数，并传递给 `HTMLFormatter`。

**文件：** `src/html_formatter.py`

`format_article()` 方法添加 `content_strategy` 参数，并传递给 `LLMClient`。

**文件：** `src/llm_client.py`

`format_article()` 和 `_create_format_prompt()` 方法都支持 `content_strategy` 参数，根据选择的策略动态生成不同的 AI 提示词。

## 示例场景

### 场景 1：格式化用户协议

**内容：** 用户隐私保护条款
**选择模式：** 🔒 严格按照原文
**原因：** 法律文件必须保持原文完整性，不能有任何改动或添加

### 场景 2：优化技术博客

**内容：** Python 编程技巧分享
**选择模式：** 📝 适度解读优化
**原因：** 可以调整逻辑顺序，添加过渡句，让文章更流畅

### 场景 3：创作营销文案

**内容：** 产品功能简介（200 字）
**选择模式：** ✨ 合理扩写丰富
**原因：** 需要补充使用场景、案例、用户反馈等，让内容更有说服力

## 注意事项

1. **默认策略是"strict"**，确保向后兼容，不会影响现有代码
2. **所有策略都必须保持核心内容完整**，不能改变原文的主要信息和观点
3. **所有策略都禁止**添加"内容完整性确认"等画蛇添足的文字
4. **所有策略都必须**包含底部标识"powered by 深表美文"
5. 选择策略时应根据实际业务需求和用户期望来决定
