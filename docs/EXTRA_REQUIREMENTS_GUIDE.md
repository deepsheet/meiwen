# 额外要求功能说明

## 功能概述

在文章美化工具中新增"额外要求"输入框，允许用户输入个性化的格式化要求，AI 会在生成 HTML 时严格遵守这些额外要求。

## 使用场景

### 示例 1：指定配色方案
```
额外要求：
- 使用蓝色系配色（#0066cc 为主色）
- 背景色要浅一些
- 标题使用深蓝色
```

### 示例 2：强调特定内容
```
额外要求：
- 重点内容用红色标注
- 关键数据加粗并添加下划线
- 重要段落添加黄色高亮背景
```

### 示例 3：调整视觉风格
```
额外要求：
- 添加更多 emoji 图标
- 卡片圆角更大一些（24px）
- 阴影效果更柔和
```

### 示例 4：优化阅读体验
```
额外要求：
- 适合移动端阅读
- 字体大小调整为 18px
- 行间距设置为 2 倍行高
- 段落之间增加更多留白
```

### 示例 5：特殊格式要求
```
额外要求：
- 所有小标题都要添加数字编号
- 列表项前必须添加 emoji
- 引用块使用灰色背景 + 左边框
- 代码片段使用等宽字体
```

### 输入框特点

- **高度适中**：4 行高度，既能输入足够内容，又不会占用过多空间
- **支持多行**：可以输入多条要求，每条一行
- **Placeholder 提示**：提供示例引导用户输入
- **可选填写**：不填则使用默认样式

## 数据传输

### 前端发送

```javascript
{
    "title": "文章标题",
    "content": "文章内容...",
    "content_strategy": "strict",
    "extra_requirements": "- 使用蓝色系配色\n- 重点内容用红色标注"
}
```

### 后端接收

```python
@app.route('/api/format-html', methods=['POST'])
def format_html_api():
    data = request.get_json()
    content = data.get('content', '')
    title = data.get('title', '').strip()
    content_strategy = data.get('content_strategy', 'strict')
    extra_requirements = data.get('extra_requirements', '')  # 接收额外要求
    
    logger.info(f"收到格式化请求，策略：{content_strategy}，额外要求：{extra_requirements}")
```

LLMClient 在构建提示词时，会将额外要求插入到显著位置：

```python
if extra_requirements and extra_requirements.strip():
    extra_requirements_text = f"""

{extra_requirements}

⚠️ 注意：以上是用户特别提出的要求，你必须在格式化时严格遵守并满足这些要求。"""
```

## 技术实现

### 前端修改

**文件：** `/static/formatter.html`
- 添加 `textarea` 输入框，id 为 `extraRequirements`
- 设置 `placeholder` 提供输入示例
- `rows="4"` 控制初始高度

**文件：** `/static/css/formater.css`
- 设置 `#extraRequirements` 样式
- `min-height: 100px` 确保足够输入空间
- `font-size: 0.95rem` 适中的字体大小

**文件：** `/static/js/formatter.js`
- 获取输入框值：`document.getElementById('extraRequirements')`
- 添加到请求体：`extra_requirements: extraRequirements`

### 后端修改

**文件：** `src/web_server.py`
- 接收参数：`extra_requirements = data.get('extra_requirements', '')`
- 记录日志：显示额外要求内容
- 传递给 formatter：作为参数传递

**文件：** `src/html_formatter.py`
- 方法签名添加参数：`extra_requirements=None`
- 日志记录额外要求
- 传递给 LLMClient

**文件：** `src/llm_client.py`
- 方法签名添加参数：`extra_requirements=None`
- 构建提示词时插入额外要求部分
- 放在策略说明之后、输出要求之前

## 优先级规则

当额外要求与其他指令冲突时，遵循以下优先级：

1. **最高优先级**：用户的额外要求
2. **次高优先级**：选择的内容处理策略
3. **基础要求**：HTML 完整性、底部标识等固定规范

### 示例：处理冲突

**用户额外要求：** "所有文字使用红色"
**系统原则：** 保持可读性

**AI 处理方式：** 
- 遵守用户要求，但会适度调整
- 可能使用深红色而非纯红，确保可读性
- 或在关键部分使用红色强调，而非全部文字

## 最佳实践建议

### ✅ 推荐的写法

1. **具体明确**
   ```
   ✅ 好：使用蓝色系配色（#667eea 为主色）
   ❌ 差：好看的颜色
   ```

2. **分条列出**
   ```
   ✅ 好：
   - 第一条要求
   - 第二条要求
   - 第三条要求
   
   ❌ 差：一大段文字混在一起
   ```

3. **可执行的要求**
   ```
   ✅ 好：添加 emoji 图标、使用圆角卡片
   ❌ 差：让它变得很酷、让人眼前一亮
   ```

4. **合理的范围**
   ```
   ✅ 好：字体大小 16-18px
   ❌ 差：字体越大越好（可能导致无法阅读）
   ```

### ⚠️ 注意事项

1. **不要与核心原则冲突**
   - 不能要求 AI 添加虚假内容
   - 不能要求移除底部标识

2. **不要过于复杂**
   - 建议 3-5 条要求
   - 太多要求可能导致效果不佳

3. **聚焦视觉和格式**
   - 适合：配色、布局、样式、emoji 等
   - 不适合：改变内容、添加观点等

## 完整示例

### 用户操作流程

1. **打开页面**：访问 `http://localhost:8009/formatter.html`

2. **输入内容**：
   ```
   文章内容：[粘贴或输入要美化的文章]
   文章标题：我的博客文章（可选）
   ```

3. **选择模式**：
   - 选择"✨ 合理扩写丰富"

4. **填写额外要求**：
   ```
   - 使用紫色系配色（#667eea 为主色）
   - 所有小标题添加数字编号和 emoji
   - 重点内容用红色标注并加粗
   - 适合移动端阅读，字体大一些
   - 添加柔和的阴影效果
   ```

5. **点击提交**："🎨 开始美化"

6. **接收结果**：AI 生成的 HTML 会严格遵守所有额外要求
